# Реализация
В рамках лабораторной работы реализовано суммирование элементов вектора последовательным алгоритмом на CPU и параллельным алгоритмом на GPU с использованием технологии CUDA.

На CPU использован последовательный алгоритм, в котором все элементы вектора обрабатываются одним потоком. 

На GPU реализация основана на parallel reduction. Входной вектор разбивается на части, и каждый поток CUDA обрабатывает отдельный элемент. Внутри каждого блока используется shared memory для выполнения суммирования значений.

Ускорение проверяется путем сравнения времени выполнения сложение, используя параллельный и последовательный алгоритм.

# Выводы
В диапазоне входных данных от 1000 до 100000 элементов использование GPU не даёт ускорения по сравнению с CPU. Это из-за неизбежных накладных расходов.

При увеличении размера входных данных до 250000 элементов виден значительный рост производительности GPU.

Таким образом, использование параллельного вычисления наиболее эффективно при обработке больших объёмов данных, 
где ускорении от параллельной обработи значительно превосходит время, затраченное на накладные расходы.
